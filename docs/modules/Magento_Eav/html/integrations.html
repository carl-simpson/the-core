Integration Overview

Magento_Eav is a foundational framework module that provides the Entity-Attribute-Value architecture used by multiple core commerce modules. It has minimal dependencies but is consumed by many critical modules for their flexible attribute systems.

Module Dependencies

Magento_Eav has minimal dependencies, allowing it to serve as a core framework component:

Dependency Magento_Store

Required for store-scoped attribute values. Each EAV value can be specific to a store view, website, or global scope.

  • StoreManagerInterface - Store context resolution
  • Store ID in value tables for scope differentiation
  • Default value inheritance from parent scopes

Dependency Magento_Theme

Provides frontend presentation infrastructure for EAV attribute rendering in layouts and templates.

  • Block rendering for attribute displays
  • Layout XML integration for admin forms
  • Theme inheritance for customizations

Consumer Modules

These modules depend on Magento_Eav for their attribute systems:

Consumer Magento_Catalog

The largest EAV consumer. Products and categories use EAV for flexible attribute management.

  • Entity Types: catalog_product, catalog_category
  • Attributes: ~200+ product attributes, ~30+ category attributes
  • Uses flat tables for frontend performance
  • Custom backend models for prices, images, etc.

Consumer Magento_Customer

Customer and address entities leverage EAV for customizable registration and account fields.

  • Entity Types: customer, customer_address
  • Attributes: ~30+ customer, ~20+ address attributes
  • Form field configuration via eav_form tables
  • Validation rules per form context

Consumer Magento_Swatches

Extends EAV to provide visual swatch functionality for configurable product options.

  • Custom frontend models for swatch rendering
  • Additional attribute metadata for swatch types
  • Color/image swatch option storage

Consumer Magento_ConfigurableProduct

Uses EAV attributes as configuration axes for product variations.

  • Super attributes for product configuration
  • Attribute-based product linking
  • Price adjustments per attribute value

Consumer Magento_CatalogSearch

Indexes EAV attributes for search functionality.

  • Searchable attribute indexing
  • Filterable attribute processing
  • Layered navigation configuration

Consumer Magento_ImportExport

Handles EAV data during product/customer import operations.

  • Attribute mapping in CSV imports
  • Multi-value attribute handling
  • Option value creation during import

Extension Points

Magento_Eav provides multiple extension mechanisms for customization:

Custom Backend Models

Backend models handle attribute data processing during save/load operations. Create custom backend models by extending Magento\Eav\Model\Entity\Attribute\Backend\AbstractBackend.

class CustomBackend extends AbstractBackend { public function beforeSave($object) { $value = $object->getData($this->getAttribute()->getAttributeCode()); // Transform/validate value before save $object->setData($this->getAttribute()->getAttributeCode(), $processedValue); return $this; } public function validate($object) { // Return true or throw LocalizedException return true; } }

Custom Frontend Models

Frontend models control how attribute values are displayed. Extend Magento\Eav\Model\Entity\Attribute\Frontend\AbstractFrontend for custom rendering.

class CustomFrontend extends AbstractFrontend { public function getValue(DataObject $object) { $value = $object->getData($this->getAttribute()->getAttributeCode()); // Format value for display (e.g., currency, date) return $this->formatValue($value); } }

Custom Source Models

Source models provide option values for select/multiselect attributes. Implement Magento\Eav\Model\Entity\Attribute\Source\SourceInterface.

class CustomSource implements SourceInterface { public function getAllOptions() { return [ ['value' => '1', 'label' => __('Option One')], ['value' => '2', 'label' => __('Option Two')], ]; } public function getOptionText($value) { // Return label for value } }

EntityManager Handlers

The EntityManager framework allows extending entity CRUD operations via AttributePool handlers.

<!-- di.xml extension --> <type name="Magento\Framework\EntityManager\Operation\AttributePool"> <arguments> <argument name="extensionActions" xsi:type="array"> <item name="eav" xsi:type="array"> <item name="Your\Entity\Type" xsi:type="array"> <item name="read" xsi:type="string">Your\Handler\Read</item> <item name="create" xsi:type="string">Your\Handler\Create</item> <item name="update" xsi:type="string">Your\Handler\Update</item> </item> </item> </argument> </arguments> </type>

Custom Entity Types

Register new EAV entity types via setup scripts for custom data models.

// Setup/InstallData.php or Setup/Patch/Data/ $eavSetup->addEntityType( 'custom_entity', [ 'entity_model' => CustomEntity::class, 'attribute_model' => Attribute::class, 'table' => 'custom_entity', 'increment_model' => null, 'additional_attribute_table' => 'custom_entity_eav_attribute', 'entity_attribute_collection' => Collection::class, ] );

Integration Best Practices

Use Service Contracts

Always interact with EAV through service contracts (repositories) rather than direct model/resource model access to ensure proper event dispatch and plugin execution.

Cache Attribute Config

Attribute configuration is cached aggressively. After programmatic attribute changes, invalidate the EAV cache tag: Magento\Eav\Model\Config::CACHE_TAG

Consider Flat Tables

For high-read entities, implement flat table indexing. EAV is flexible but slow for complex queries. Catalog uses flat tables for frontend.

Batch Operations

For bulk attribute updates, use the EntityManager's batch operations or direct SQL with proper cache invalidation rather than iterating repositories.
